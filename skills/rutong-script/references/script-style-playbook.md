# Script Style Playbook

## 总原则

- 讲稿和 deck 分开，但保持页码同步。
- 讲稿要能直接说出口，少长句。
- 默认中文，中英文关键词保留。
- 不要过度 polished，不要 AI marketing 味太重。
- 受众不同，表达不同：boss 看价值和 ask；BI/DI 看协作与落地；普通用户看怎么开始和有什么帮助。

## 受众适配

| 受众 | 重点 | 语气 |
|---|---|---|
| Boss | 进展、价值、风险、ask | 结论前置、少过程 |
| BI / DI | 共同语言、交付边界、协作机制 | 工作化、可执行 |
| AI Eco participant | 从功能视角升级到用户任务视角 | 共创、低压力 |
| 非技术用户 | 入口、步骤、支持渠道 | 简单、具体 |
| 培训学员 | 概念 + 演示 + 上手路径 | 温和、可跟上 |

## 常见任务

- `page-level notes`：按页给讲稿，不要重写整篇。
- `audience adaptation`：标出 keep / rewrite / de-emphasize。
- `interaction prompts`：给页码、问题、低压力说法。
- `shorten`：保留主线和例子，删解释性铺垫。
