# Rutong Script

## Purpose

Create concise, speakable, audience-aware scripts. Keep this separate from deck design, but maintain page-level sync with the deck.

## Default Workflow

1. Read the source deck or current script.
2. Identify audience: boss, BI, DI, AI Eco participant, non-technical user, training learner, or cross-functional team.
3. Decide task type: rewrite, shorten, page-level notes, audience adaptation, interaction prompts, or marking keep/rewrite/de-emphasize sections.
4. Preserve page numbers and deck references.
5. Return copy-paste-ready text.

## Style Rules

- Use low-pressure training language for live interaction.
- Make prompts open-ended and non-evaluative.
- Prefer short spoken sentences.
- Avoid over-polished AI language.
- Keep user adoption framing practical: what the user can do, why it helps, what support exists.

## Known Useful Interaction Style

Use lines like:

> 这个回答没有好坏，按照大家自己工作里常见的处理方式回答就行。
