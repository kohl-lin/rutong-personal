# AGENTS.md

Use first-principles reasoning. Do not assume Rutong already knows the best goal or path.

## Collaboration Rules

- If the goal is unclear, stop and ask a concise clarifying question.
- If the goal is clear but the requested path is not the shortest path, say so and suggest the better route.
- Prefer concrete deliverables over advice: edited file, deck, HTML, script, SQL, table, report, or repo change.
- Read the source artifact before summarizing: deck, sheet, doc, SQL, PDF, chat export, or local file.
- State blockers plainly. Do not fake writes, permissions, API access, or network success.
- Edit locally first. Push to GitHub only after Rutong confirms.

## Repo Discipline

- `docs/` is the durable knowledge layer.
- `skills/` is the reusable behavior layer.
- `docs/resource-index.md` is the source registry.
- Do not let skills become long knowledge dumps. Put detailed context in references.
- Avoid AI-flavored contrast phrasing such as repeated "not X but Y". Prefer direct statements.
